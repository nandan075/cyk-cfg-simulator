# CFG Membership Checker — CYK Algorithm

## Theory of Computation — Mini Project

A Python application that determines whether a given string belongs to the language
generated by a **Context-Free Grammar (CFG)** using the **CYK (Cocke-Younger-Kasami) Algorithm**.

---

## 📋 Features

- **Grammar Input**: Enter CFG productions in a simple, readable format
- **CNF Conversion**: Automatically converts CFG to Chomsky Normal Form (5-step process)
- **CYK Algorithm**: Implements the CYK dynamic programming algorithm
- **Visual CYK Table**: Displays the parsing table with color-coded cells
- **Step-by-Step Execution**: Shows detailed breakdown of each algorithm step
- **Sample Grammars**: Pre-loaded example grammars for quick testing
- **Input Validation**: Handles malformed grammars with clear error messages
- **Multiple Modes**: GUI (Tkinter), CLI, and automated test suite

## 📘 Grammar Input Format

Each line represents productions for one non-terminal:

```
S -> aSb | ab
A -> aA | a
```

### Conventions

| Symbol                            | Meaning                           |
| --------------------------------- | --------------------------------- |
| Uppercase letters (`S`, `A`, `B`) | Non-terminals                     |
| Lowercase letters (`a`, `b`, `c`) | Terminals                         |
| `ε` or `epsilon`                  | Empty string (epsilon)            |
| `\|`                              | Separates alternative productions |
| `->`                              | Separates LHS from RHS            |

## 🧮 Algorithm Overview

### 1. CFG → CNF Conversion (5 Steps)

| Step       | Description                                                     |
| ---------- | --------------------------------------------------------------- |
| **Step 1** | Add new start symbol `S₀ → S`                                   |
| **Step 2** | Eliminate ε-productions                                         |
| **Step 3** | Eliminate unit productions (`A → B`)                            |
| **Step 4** | Replace terminals in mixed bodies (`A → aBc` → `A → T_a B T_c`) |
| **Step 5** | Break long productions into binary chains                       |

### 2. CYK Algorithm

- **Input**: Grammar in CNF + input string `w` of length `n`
- **Table**: `T[i][j]` = set of non-terminals that derive substring `w[i..j]`
- **Base case**: Fill diagonal — `T[i][i] = {A : A → w[i] ∈ G}`
- **Induction**: For each length `ℓ = 2..n`, fill `T[i][j]`:
  - For each split point `k`: if `A → BC`, `B ∈ T[i][k]`, `C ∈ T[k+1][j]` → add `A`
- **Result**: String accepted ⟺ `S ∈ T[0][n-1]`

## 📚 Theory Background

The **CFG Membership Problem** (also called the **word problem for CFGs**) asks:

> Given a CFG `G` and a string `w`, is `w ∈ L(G)`?

This is a **decidable** problem, solvable in **O(n³·|G|)** time using the CYK algorithm,
where `n = |w|` is the length of the input string and `|G|` is the size of the grammar.

The CYK algorithm requires the grammar to be in **Chomsky Normal Form (CNF)**, where every
production is either:

- `A → BC` (two non-terminals), or
- `A → a` (single terminal)
